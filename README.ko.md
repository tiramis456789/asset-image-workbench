# Asset Image Workbench

[English README](README.md)

Asset Image Workbench는 디스크에 실제 변경을 적용하기 전에 이미지 파일을 검토하고, 정렬하고, 이름을 바꾸고, 폴더 구조를 재구성할 수 있는 브라우저 기반 작업 도구입니다.

이 프로젝트는 특히 프롬프트 정보나 EXIF 유사 메타데이터가 중요한 이미지 컬렉션 워크플로를 위해 만들어졌습니다. 실제 파일에 손대기 전까지는 변경 사항을 가상 상태로 유지하므로, 적용 결과를 먼저 검토한 뒤 마지막에만 실제 폴더에 반영할 수 있습니다.

## 주요 기능

- 필름스트립과 멀티 뷰 비교를 지원하는 이미지 뷰어
- 핀 고정 및 핀만 보기 흐름
- 가상 이름 변경 및 가상 폴더 이동 워크플로
- 실제 파일 반영 전 미리보기
- `log.txt` 기록을 남기는 실제 반영 단계
- EXIF 확인 및 태그 기반 후보 작업 흐름
- 구조화된 파일명을 만들 수 있는 레고식 템플릿 빌더

## 현재 상태

- 이름 변경, 이동, EXIF 확인, 실제 반영 핵심 흐름이 구현되어 있습니다
- 실제 파일 반영은 쓰기 가능한 폴더 열기 흐름으로 연 폴더에서만 사용할 수 있습니다
- 데스크톱 패키징을 사용할 수 있습니다
- 자동화 테스트는 이름 변경 규칙, 반영 동작, 이동 규칙, 태그 편집, EXIF 필터를 다룹니다

## 기술 스택

- React 18
- TypeScript
- Vite
- Tailwind CSS
- Radix UI / shadcn 스타일 UI 컴포넌트
- Vitest

## 요구 사항

- Node.js 20 이상 권장
- 쓰기 가능한 폴더 접근을 지원하는 최신 Chromium 기반 브라우저

실제 반영 흐름은 File System Access API에 의존합니다. 실사용 기준으로는 Chromium 계열 브라우저에서 가장 잘 동작합니다.

## 설치

```sh
npm install
```

## 실행

```sh
npm run dev
```

터미널에 표시되는 로컬 Vite URL을 열면 됩니다.

## 빌드

```sh
npm run build
```

## 데스크톱 미리보기

```sh
npm run desktop:preview
```

이 명령은 먼저 Vite 앱을 빌드한 뒤, 로컬 프로덕션 번들을 대상으로 Electron 셸을 실행합니다.

## 포터블 EXE 빌드

```sh
npm run build:portable
```

Windows 포터블 실행 파일은 아래 위치에 생성됩니다.

```text
release/AssetImageWorkbench-<version>-portable.exe
```

이 빌드는 설치 없이 내려받아 바로 실행할 수 있는 포터블 배포 형식입니다.
이 프로젝트는 오래 설치해 두고 쓰는 데스크톱 앱보다는 가볍게 공유하고 필요할 때 실행하는 커뮤니티 유틸에 가깝기 때문에, 포터블 빌드를 권장 배포 형식으로 사용합니다.

## 일반적인 사용 흐름

1. 이미지를 불러오거나 쓰기 가능한 폴더를 엽니다.
2. 뷰어, 필름스트립, 멀티 뷰 레이아웃에서 이미지를 검토합니다.
3. 가상 작업 공간에서 핀 고정, 필터링, 이름 변경, 이동을 수행합니다.
4. 예정된 이름 변경과 이동 내역을 확인합니다.
5. 미리보기 결과가 괜찮을 때만 실제 폴더에 반영합니다.

## 실제 반영 관련 메모

- 일반 파일로 추가한 이미지는 가상으로 검토하고 편집할 수 있지만, 실제 반영은 쓰기 가능한 폴더 원본이 필요합니다.
- 앱은 반영 전에 대상 파일 충돌을 검사합니다.
- 부분 실패가 발생해도 성공한 변경은 유지되며, 실패 내용은 로그에 남습니다.
- 실제 반영 시 쓰기 가능한 루트 폴더에 `log.txt`가 생성됩니다.

## 프로젝트 범위

이 프로젝트는 호스팅형 SaaS 제품이 아니라 개인 및 커뮤니티 사용을 목표로 합니다. 초점은 서버 기능이나 계정 시스템이 아니라, 이미지 중심 작업 흐름에서 실용적인 일괄 정리 도구를 제공하는 데 있습니다.

## 제한사항

- 쓰기 가능한 폴더 기능은 브라우저 지원 범위가 제한적입니다
- 일부 UI 문구는 아직 정리가 더 필요합니다
- 생성된 UI 래퍼 파일들 때문에 기존 Fast Refresh 린트 경고가 남아 있습니다
- 패키징된 데스크톱 빌드는 현재 Electron 기본 앱 아이콘을 사용합니다

## 패키징 및 보안 메모

- 권장 배포 형식은 Windows 포터블 실행 파일입니다. 이 프로젝트는 가끔씩 로컬에서 사용하는 저마찰 커뮤니티 유틸을 목표로 합니다.
- Electron 데스크톱 빌드는 `contextIsolation: true`와 `nodeIntegration: false`를 유지합니다.
- 패키징된 빌드에서 `sandbox: true`를 켜면 Windows 포터블 실행 테스트에서 검은 화면 회귀가 발생했기 때문에, 현재 렌더러 샌드박스는 비활성화된 상태입니다.
- 이것은 장기적으로 이상적인 보안 상태라기보다, 현재 릴리스의 실행 호환성을 우선한 의도적인 트레이드오프입니다.
- 이후 샌드박스 하드닝 작업은 로컬 개발 환경이나 정적 검증만으로 판단하지 말고, 패키징된 실행 파일 기준으로 검증해야 합니다.
- 현재 포터블 릴리스는 공유 전 VirusTotal로 점검했으며, 분석을 완료한 엔진 기준으로 악성 탐지는 확인되지 않았습니다.
- 일부 VirusTotal 엔진은 시간 초과 또는 파일 형식 미지원 결과를 반환했으며, 이는 악성 탐지가 아니라 미완료 분석으로 봐야 합니다.
